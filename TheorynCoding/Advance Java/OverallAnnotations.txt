# ЁЯФ╡ **Cycle 0 тАУ Core Java Annotations (JDK Built-in + Meta)**

---

## ЁЯУЦ **Concept**

Core Java defines a **small set of built-in annotations** and **meta-annotations**.
They are the foundation for building and using annotations in Java.

* **Built-in Annotations:**
  `@Override`, `@Deprecated`, `@SuppressWarnings`, `@SafeVarargs`, `@FunctionalInterface`

* **Meta Annotations:**
  `@Target`, `@Retention`, `@Documented`, `@Inherited`, `@Repeatable`, `@Native`

---

## ЁЯзй **Example**

```java
import java.lang.annotation.*;

class Parent {
    @Deprecated
    void oldMethod() {
        System.out.println("Old method - Deprecated");
    }
}

class Child extends Parent {
    @Override
    void oldMethod() {   // compiler checks correctness
        System.out.println("Overridden method");
    }

    @SuppressWarnings("unchecked")
    void testList() {
        java.util.List list = new java.util.ArrayList(); // unchecked warning suppressed
        list.add("Hello");
    }
}

// Custom annotation using meta-annotations
@Retention(RetentionPolicy.RUNTIME)
@Target(ElementType.METHOD)
@interface MyAnnotation {
    String value();
}

class Demo {
    @MyAnnotation("Testing custom annotation")
    public void show() {}
}
```

---

## ЁЯОп **Interview Q\&A**

### тЬЕ **Simple**

1. What does `@Override` do?
   ЁЯСЙ Ensures a method overrides a superclass method, otherwise compiler error.

2. What is the use of `@Deprecated`?
   ЁЯСЙ Marks code as outdated, compiler gives a warning when used.

---

### ЁЯдФ **Tricky**

3. Will the code compile if you put `@Override` on a method that does not override?
   ЁЯСЙ No. Compiler throws error тЖТ ensures correctness.

4. What is the role of `@SuppressWarnings`?
   ЁЯСЙ Used to suppress compiler warnings (like unchecked generics, deprecations).

---

### ЁЯФе **Tough**

5. Difference between `@Retention` and `@Target`?
   ЁЯСЙ

* `@Retention` тЖТ lifecycle (SOURCE, CLASS, RUNTIME).
* `@Target` тЖТ specifies where annotation can be applied (method, class, field, etc.).

6. Can annotations have default values?
   ЁЯСЙ Yes, using `default` inside annotation definition.

---

### ЁЯПв **MNC-Style**

7. How to create and use a custom annotation at runtime?
   ЁЯСЙ By defining annotation with `@Retention(RetentionPolicy.RUNTIME)` and reading via **Reflection**.

```java
public class Test {
    public static void main(String[] args) throws Exception {
        var method = Demo.class.getMethod("show");
        MyAnnotation anno = method.getAnnotation(MyAnnotation.class);
        System.out.println(anno.value());  // Prints: Testing custom annotation
    }
}
```

8. What is `@Inherited`?
   ЁЯСЙ Makes custom annotations **inheritable by subclasses** if applied at class level.

# ЁЯЯв **Cycle 1 тАУ Spring Core Annotations (Bean Definition & DI)**

---

## ЁЯУЦ **Concept**

Spring provides annotations to **define beans** and manage **dependency injection**.
These replace XML configuration with annotation-based configuration.

* **Stereotype Annotations (Bean definition):**

  * `@Component` тЖТ generic bean
  * `@Service` тЖТ service layer bean (specialized `@Component`)
  * `@Repository` тЖТ DAO (database access bean) with persistence exception translation
  * `@Controller` тЖТ MVC controller (used in web apps)
  * `@RestController` тЖТ `@Controller + @ResponseBody` (REST APIs)

* **Dependency Injection Annotations:**

  * `@Autowired` тЖТ injects bean by type
  * `@Qualifier` тЖТ resolves ambiguity between multiple beans of same type
  * `@Primary` тЖТ marks one bean as the default choice
  * `@Value` тЖТ injects values from properties file or literals
  * `@Lazy` тЖТ bean created only when needed
  * `@Scope` тЖТ defines bean lifecycle (`singleton`, `prototype`, etc.)

---

## ЁЯзй **Example**

```java
import org.springframework.beans.factory.annotation.*;
import org.springframework.context.annotation.*;
import org.springframework.stereotype.*;

// Service Bean
@Service
class UserService {
    public String getUser() {
        return "NikhiTech";
    }
}

// DAO Bean
@Repository
class UserRepository {
    public String getData() {
        return "User data from DB";
    }
}

// Controller Bean
@RestController
class UserController {

    @Autowired   // dependency injection
    private UserService userService;

    @Autowired
    private UserRepository userRepository;

    @GetMapping("/user")
    public String getUserInfo() {
        return userService.getUser() + " | " + userRepository.getData();
    }
}

@Configuration
@ComponentScan(basePackages = "com.example")
class AppConfig { }
```

---

## ЁЯОп **Interview Q\&A**

### тЬЕ **Simple**

1. What is the difference between `@Component` and `@Service`?
   ЁЯСЙ Both define beans, but `@Service` is semantic (used for service layer).

2. What does `@Autowired` do?
   ЁЯСЙ It injects a bean **by type** from the Spring container.

---

### ЁЯдФ **Tricky**

3. What happens if two beans of the same type exist and you use `@Autowired`?
   ЁЯСЙ Spring throws **NoUniqueBeanDefinitionException** unless resolved with `@Qualifier` or `@Primary`.

4. WhatтАЩs the difference between `@Controller` and `@RestController`?
   ЁЯСЙ `@RestController` = `@Controller + @ResponseBody`. It returns JSON/XML directly instead of a view.

---

### ЁЯФе **Tough**

5. How does `@Repository` differ from `@Component`?
   ЁЯСЙ Both register beans, but `@Repository` also provides **automatic persistence exception translation** (converts JPA/Hibernate exceptions into SpringтАЩs `DataAccessException`).

6. What happens if a bean is annotated with `@Lazy`?
   ЁЯСЙ Bean is created **only when first requested**, not at container startup.

---

### ЁЯПв **MNC-Style**

7. How does Spring resolve ambiguity when multiple beans of same type exist?
   ЁЯСЙ

* Use `@Qualifier("beanName")`
* Use `@Primary` on one bean
* Programmatic injection with `@Autowired(required=false)`

8. Can `@Autowired` inject primitive types like `String` or `int`?
   ЁЯСЙ Yes, but it works with `@Value` or property injection (`application.properties`). Example:

```java
@Value("${app.name:DefaultApp}")
private String appName;
```

# ЁЯЯа **Cycle 2 тАУ Spring Boot Annotations**

---

## ЁЯУЦ **Concept**

Spring Boot provides special annotations to simplify configuration, auto-wiring, and application bootstrapping.

* **Main Bootstrapping**

  * `@SpringBootApplication` тЖТ combo of `@Configuration`, `@EnableAutoConfiguration`, `@ComponentScan`.
  * `@EnableAutoConfiguration` тЖТ lets Spring Boot configure beans automatically based on classpath dependencies.
  * `@ComponentScan` тЖТ scans packages for beans (`@Component`, `@Service`, etc.).
  * `@Configuration` тЖТ marks a class for Java-based Spring configuration.
  * `@Bean` тЖТ defines a bean manually inside a `@Configuration` class.

* **Conditional & Profiles**

  * `@Profile` тЖТ activates beans based on active environment profile (`dev`, `test`, `prod`).
  * `@ConditionalOnProperty` тЖТ loads bean if a property exists in `application.properties`.
  * `@ConditionalOnMissingBean` тЖТ loads bean only if not already defined.

* **Configuration Binding**

  * `@ConfigurationProperties` тЖТ binds values from `application.properties` or `application.yml` to a POJO.

---

## ЁЯзй **Example**

```java
import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;
import org.springframework.boot.context.properties.ConfigurationProperties;
import org.springframework.context.annotation.*;

@SpringBootApplication
public class DemoApplication {
    public static void main(String[] args) {
        SpringApplication.run(DemoApplication.class, args);
    }
}

@Configuration
class MyConfig {
    @Bean
    public String myBean() {
        return "Hello Bean";
    }
}

@ConfigurationProperties(prefix = "app")
class AppProperties {
    private String name;
    private String version;
    // getters & setters
}
```

`application.properties`

```
app.name=MySpringBootApp
app.version=1.0
```

---

## ЁЯОп **Interview Q\&A**

### тЬЕ **Simple**

1. What does `@SpringBootApplication` do?
   ЁЯСЙ ItтАЩs a shortcut for `@Configuration + @EnableAutoConfiguration + @ComponentScan`.

2. What is `@Bean` used for?
   ЁЯСЙ To manually declare a Spring bean inside a `@Configuration` class.

---

### ЁЯдФ **Tricky**

3. Difference between `@Configuration` and `@Component`?
   ЁЯСЙ

* `@Configuration` тЖТ used for **bean definitions** (`@Bean`).
* `@Component` тЖТ generic stereotype for auto-detection.

4. What happens if two beans of same type are created with `@Bean`?
   ЁЯСЙ It causes **NoUniqueBeanDefinitionException** unless one is marked with `@Primary` or resolved with `@Qualifier`.

---

### ЁЯФе **Tough**

5. How does `@EnableAutoConfiguration` work internally?
   ЁЯСЙ Uses **Spring Factories** (`spring.factories`) + **classpath scanning** to auto-configure beans depending on available libraries (e.g., `spring-boot-starter-data-jpa`).

6. What is the difference between `@Value` and `@ConfigurationProperties`?
   ЁЯСЙ

* `@Value` тЖТ injects **single property value**.
* `@ConfigurationProperties` тЖТ maps a **group of properties** into a POJO.

---

### ЁЯПв **MNC-Style**

7. How do you load a bean only in the `dev` environment?
   ЁЯСЙ

```java
@Profile("dev")
@Component
public class DevOnlyBean { }
```

Run app with `--spring.profiles.active=dev`.

8. How do you create a bean conditionally only if a property exists?
   ЁЯСЙ

```java
@ConditionalOnProperty(name = "feature.enabled", havingValue = "true")
@Bean
public FeatureService featureService() {
    return new FeatureService();
}
```

# ЁЯФ┤ **Cycle 3 тАУ Spring Web / MVC Annotations**

---

## ЁЯУЦ **Concept**

Spring Web MVC provides annotations for building **web applications & REST APIs**.
They control **request mapping, request/response handling, data binding, and exception handling**.

* **Controller Layer**

  * `@Controller` тЖТ MVC controller returning view
  * `@RestController` тЖТ REST API controller (JSON/XML response by default)

* **Request Mapping**

  * `@RequestMapping` тЖТ maps URL & HTTP method
  * `@GetMapping`, `@PostMapping`, `@PutMapping`, `@DeleteMapping`, `@PatchMapping` тЖТ specialized shortcuts

* **Request Data Handling**

  * `@RequestParam` тЖТ extracts query parameters
  * `@PathVariable` тЖТ extracts URI path variables
  * `@RequestBody` тЖТ binds request body (JSON тЖТ Java object)
  * `@ResponseBody` тЖТ returns object directly as response body
  * `@RequestHeader` тЖТ extracts header values
  * `@CookieValue` тЖТ extracts cookies
  * `@ModelAttribute` тЖТ binds form data to an object
  * `@SessionAttributes` тЖТ store model attributes in HTTP session

* **Exception Handling**

  * `@ExceptionHandler` тЖТ handle exceptions inside controller
  * `@ControllerAdvice` тЖТ global exception handling

* **Cross-Origin Support**

  * `@CrossOrigin` тЖТ enables CORS (Cross-Origin Resource Sharing)

---

## ЁЯзй **Example**

```java
import org.springframework.web.bind.annotation.*;

@RestController
@RequestMapping("/api/users")
public class UserController {

    // GET with query param
    @GetMapping
    public String getUser(@RequestParam String name) {
        return "Hello " + name;
    }

    // GET with path variable
    @GetMapping("/{id}")
    public String getUserById(@PathVariable int id) {
        return "User ID: " + id;
    }

    // POST with JSON body
    @PostMapping
    public String createUser(@RequestBody User user) {
        return "User created: " + user.getName();
    }

    // Handle exception
    @ExceptionHandler(RuntimeException.class)
    public String handleError(RuntimeException ex) {
        return "Error: " + ex.getMessage();
    }
}

class User {
    private String name;
    // getters & setters
}
```

---

## ЁЯОп **Interview Q\&A**

### тЬЕ **Simple**

1. Difference between `@Controller` and `@RestController`?
   ЁЯСЙ `@RestController = @Controller + @ResponseBody` тЖТ directly returns JSON/XML instead of view.

2. WhatтАЩs the difference between `@RequestParam` and `@PathVariable`?
   ЁЯСЙ

* `@RequestParam` тЖТ query string param (`/users?id=10`)
* `@PathVariable` тЖТ part of URI path (`/users/10`)

---

### ЁЯдФ **Tricky**

3. Can you use multiple `@RequestMapping` on one method?
   ЁЯСЙ Yes, you can map multiple URLs to one method:

```java
@RequestMapping({"/home", "/index"})
```

4. Difference between `@RequestBody` and `@ModelAttribute`?
   ЁЯСЙ

* `@RequestBody` тЖТ for JSON/XML payload тЖТ object mapping.
* `@ModelAttribute` тЖТ for form data binding (used in MVC forms).

---

### ЁЯФе **Tough**

5. How does Spring decide content type (JSON, XML, HTML) in a `@RestController`?
   ЁЯСЙ Using **Content Negotiation**:

* Checks **Accept header** from client (`application/json`, `application/xml`).
* Uses **HttpMessageConverters** internally to serialize/deserialize.

6. WhatтАЩs the difference between `@ExceptionHandler` and `@ControllerAdvice`?
   ЁЯСЙ

* `@ExceptionHandler` тЖТ handles exceptions **inside one controller**.
* `@ControllerAdvice` тЖТ global exception handling across all controllers.

---

### ЁЯПв **MNC-Style**

7. Suppose two methods are mapped to `/user/{id}` тАУ one expecting `int id`, another expecting `String id`. What happens?
   ЁЯСЙ Ambiguity тЖТ Spring throws **IllegalStateException** (ambiguous mapping).

8. How do you enable a REST endpoint to be called from another domain (CORS issue)?
   ЁЯСЙ Use `@CrossOrigin`:


# ЁЯЯг **Cycle 4 тАУ Spring AOP (Aspect-Oriented Programming) Annotations**

---

## ЁЯУЦ **Concept**

Spring AOP (Aspect-Oriented Programming) is used to handle **cross-cutting concerns** (logging, security, transactions) separately from business logic.
Annotations define **where** and **when** extra behavior (advice) should run.

* **Aspect Declaration**

  * `@Aspect` тЖТ marks a class as an Aspect.
  * `@EnableAspectJAutoProxy` тЖТ enables AOP proxying in Spring.

* **Advice Types**

  * `@Before` тЖТ run before method execution.
  * `@After` тЖТ run after method execution (always, success or failure).
  * `@AfterReturning` тЖТ run only after successful return.
  * `@AfterThrowing` тЖТ run if method throws exception.
  * `@Around` тЖТ run before and after method (can control execution).

* **Join Point Expressions**

  * `@Pointcut` тЖТ reusable expression for method interception.

---

## ЁЯзй **Example**

```java
import org.aspectj.lang.annotation.*;
import org.aspectj.lang.ProceedingJoinPoint;
import org.springframework.stereotype.Component;

@Aspect
@Component
public class LoggingAspect {

    @Before("execution(* com.example.service.*.*(..))")
    public void logBefore() {
        System.out.println("Before method execution");
    }

    @AfterReturning("execution(* com.example.service.*.*(..))")
    public void logAfterReturn() {
        System.out.println("After successful execution");
    }

    @AfterThrowing("execution(* com.example.service.*.*(..))")
    public void logAfterException() {
        System.out.println("After exception thrown");
    }

    @Around("execution(* com.example.service.*.*(..))")
    public Object logAround(ProceedingJoinPoint joinPoint) throws Throwable {
        System.out.println("Around before: " + joinPoint.getSignature());
        Object result = joinPoint.proceed(); // execute target method
        System.out.println("Around after: " + joinPoint.getSignature());
        return result;
    }
}
```

---

## ЁЯОп **Interview Q\&A**

### тЬЕ **Simple**

1. What is `@Aspect` in Spring?
   ЁЯСЙ It marks a class as containing cross-cutting logic (advice).

2. WhatтАЩs the difference between `@Before` and `@Around`?
   ЁЯСЙ `@Before` runs only before execution, while `@Around` runs both before and after, and can even skip method execution.

---

### ЁЯдФ **Tricky**

3. Can you use multiple advices (`@Before`, `@After`) on the same method?
   ЁЯСЙ Yes, they are executed in defined order.

4. What does `execution(* com.example.service.*.*(..))` mean?
   ЁЯСЙ A pointcut expression тЖТ intercepts **any method inside package `com.example.service`**.

---

### ЁЯФе **Tough**

5. How does Spring AOP work internally?
   ЁЯСЙ Uses **dynamic proxies (JDK proxies or CGLIB)** to wrap beans and apply advice at runtime.

6. Difference between `@After` and `@AfterReturning`?
   ЁЯСЙ

* `@After` тЖТ runs always (success or exception).
* `@AfterReturning` тЖТ runs only on successful return.

---

### ЁЯПв **MNC-Style**

7. If a method is private, can AOP advice intercept it?
   ЁЯСЙ No тЭМ. Spring AOP works only on **public methods of Spring beans**.

8. How would you implement performance monitoring for methods using AOP?
   ЁЯСЙ Use `@Around` advice to measure start & end time:

```java
@Around("execution(* com.example..*(..))")
public Object trackTime(ProceedingJoinPoint pjp) throws Throwable {
    long start = System.currentTimeMillis();
    Object result = pjp.proceed();
    long end = System.currentTimeMillis();
    System.out.println("Execution time: " + (end - start) + "ms");
    return result;
}
```

# ЁЯЯб **Cycle 5 тАУ Spring Data JPA & Hibernate Annotations**

---

## ЁЯУЦ **Concept**

Spring Data JPA and Hibernate use annotations to map **Java objects тЖФ Database tables** and to define queries/transactions.

### **Entity Mapping**

* `@Entity` тЖТ marks a class as JPA entity.
* `@Table` тЖТ maps entity to a specific table.
* `@Id` тЖТ primary key.
* `@GeneratedValue` тЖТ auto-generate PK (IDENTITY, SEQUENCE, AUTO).
* `@Column` тЖТ map field to column, customize name/type.
* `@Transient` тЖТ ignore field (not persisted).
* `@Embedded`, `@Embeddable` тЖТ value object embedding.

### **Relationships**

* `@OneToOne`, `@OneToMany`, `@ManyToOne`, `@ManyToMany`
* `@JoinColumn`, `@JoinTable` тЖТ customize join mappings.
* `mappedBy` тЖТ defines relationship ownership.

### **Spring Data JPA**

* `@Repository` тЖТ DAO marker.
* `@Query` тЖТ custom JPQL/SQL queries.
* `@Modifying` тЖТ for update/delete queries.
* `@Transactional` тЖТ transaction boundaries.
* `@EnableJpaRepositories` тЖТ enable repository scanning.

### **Auditing**

* `@CreatedDate`, `@LastModifiedDate` тЖТ auto populate auditing fields.

---

## ЁЯзй **Example**

```java
import jakarta.persistence.*;
import org.springframework.data.jpa.repository.*;
import org.springframework.stereotype.Repository;
import org.springframework.transaction.annotation.Transactional;
import java.util.List;

@Entity
@Table(name = "users")
public class User {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(nullable = false)
    private String name;

    @OneToMany(mappedBy = "user")
    private List<Order> orders;
}

@Entity
@Table(name = "orders")
class Order {
    @Id @GeneratedValue
    private Long id;

    @ManyToOne
    @JoinColumn(name = "user_id")
    private User user;
}

@Repository
interface UserRepository extends JpaRepository<User, Long> {
    @Query("SELECT u FROM User u WHERE u.name = :name")
    List<User> findByName(String name);

    @Modifying
    @Transactional
    @Query("DELETE FROM User u WHERE u.name = :name")
    void deleteByName(String name);
}
```

## ЁЯОп **Interview Q\&A**

### тЬЕ **Simple**

1. What does `@Entity` do?
   ЁЯСЙ Marks a Java class as a JPA entity (mapped to DB table).

2. Difference between `@Id` and `@GeneratedValue`?
   ЁЯСЙ `@Id` тЖТ primary key, `@GeneratedValue` тЖТ strategy to auto-generate ID.

---

### ЁЯдФ **Tricky**

3. Difference between `@OneToMany` and `@ManyToOne`?
   ЁЯСЙ

* `@OneToMany` тЖТ parent has list of children.
* `@ManyToOne` тЖТ child has reference to parent.

4. What happens if you forget `mappedBy` in `@OneToMany`?
   ЁЯСЙ Hibernate creates a **join table** instead of foreign key mapping.

---

### ЁЯФе **Tough**

5. How does `@Transactional` work?
   ЁЯСЙ Uses **AOP proxies** + transaction manager to manage commit/rollback automatically.

6. Difference between `@Query` and method name queries (`findByName`)?
   ЁЯСЙ

* `@Query` тЖТ custom query (JPQL/SQL).
* Derived method тЖТ auto-generated query from method name.

---

### ЁЯПв **MNC-Style**

7. If you mark a field with `@Transient`, what happens?
   ЁЯСЙ Field is ignored in DB, but still available in Java object (useful for calculated values).

8. Suppose you have:

```java
@OneToOne(cascade = CascadeType.ALL, orphanRemoval = true)
```

ЁЯСЙ What happens if you delete the parent?
ЁЯСЙ Child entity is also **deleted automatically** (cascading + orphan removal).

# тЪл **Cycle 6 тАУ Spring Security Annotations**

---

## ЁЯУЦ **Concept**

Spring Security uses annotations for **authentication, authorization, and configuration**.
They help secure endpoints, restrict access, and define security rules declaratively.

### **Configuration**

* `@EnableWebSecurity` тЖТ enables Spring SecurityтАЩs web security.
* `@Configuration` тЖТ marks class as security config.

### **Method-Level Security**

* `@PreAuthorize("expression")` тЖТ runs **before** method executes (e.g., role check).
* `@PostAuthorize("expression")` тЖТ runs **after** method executes (can check return value).
* `@Secured("ROLE_X")` тЖТ restricts method access to specific roles.
* `@RolesAllowed("ROLE_X")` тЖТ same as `@Secured`, JSR-250 standard.

### **Testing**

* `@WithMockUser` тЖТ used in Spring Security tests to mock authenticated users.

---

## ЁЯзй **Example**

```java
import org.springframework.security.access.annotation.Secured;
import org.springframework.security.access.prepost.*;
import org.springframework.security.config.annotation.web.configuration.*;
import org.springframework.web.bind.annotation.*;

@EnableWebSecurity
public class SecurityConfig {
    // configure security rules here (e.g., HttpSecurity)
}

@RestController
@RequestMapping("/api")
public class SecureController {

    @PreAuthorize("hasRole('ADMIN')")
    @GetMapping("/admin")
    public String adminOnly() {
        return "Admin Access";
    }

    @Secured("ROLE_USER")
    @GetMapping("/user")
    public String userOnly() {
        return "User Access";
    }

    @PostAuthorize("returnObject.owner == authentication.name")
    @GetMapping("/profile")
    public Profile getProfile() {
        return new Profile("NikhiTech"); // check ownership after method runs
    }
}

class Profile {
    String owner;
    Profile(String o) { owner = o; }
    public String getOwner() { return owner; }
}
```

---

## ЁЯОп **Interview Q\&A**

### тЬЕ **Simple**

1. What does `@EnableWebSecurity` do?
   ЁЯСЙ It enables Spring SecurityтАЩs web security support and provides the configuration base.

2. Difference between `@Secured` and `@PreAuthorize`?
   ЁЯСЙ

* `@Secured` тЖТ simple role-based checks.
* `@PreAuthorize` тЖТ supports **SpEL (Spring Expression Language)** for advanced conditions.

---

### ЁЯдФ **Tricky**

3. Can we use both `@Secured` and `@PreAuthorize` together?
   ЁЯСЙ Yes, but you must enable method security with `@EnableGlobalMethodSecurity`.

4. Difference between `@PreAuthorize` and `@PostAuthorize`?
   ЁЯСЙ

* `@PreAuthorize` тЖТ checks **before method runs**.
* `@PostAuthorize` тЖТ checks **after method runs**, even based on return value.

---

### ЁЯФе **Tough**

5. How does Spring Security enforce role checks internally?
   ЁЯСЙ Uses **AOP proxies** to intercept method calls, then checks the **Authentication object** in SecurityContext.

6. Why is `@RolesAllowed` sometimes preferred over `@Secured`?
   ЁЯСЙ Because `@RolesAllowed` is **JSR-250 standard** тЖТ portable across frameworks.

---

### ЁЯПв **MNC-Style**

7. How do you secure a REST API so only a specific user can access their own resource?
   ЁЯСЙ Use `@PreAuthorize("#username == authentication.name")`

```java
@PreAuthorize("#username == authentication.name")
@GetMapping("/profile/{username}")
public String getProfile(@PathVariable String username) {
    return "Profile of " + username;
}
```

8. In testing, how do you simulate a logged-in user with roles?
   ЁЯСЙ Use `@WithMockUser(username="testUser", roles={"USER"})` in JUnit tests.

# ЁЯЯд **Cycle 7 тАУ Testing Annotations (JUnit, Mockito, TestNG)**

---

## ЁЯУЦ **Concept**

Testing frameworks provide annotations for **unit testing, mocking, and lifecycle management**.
The most common are **JUnit 4, JUnit 5, Mockito, and TestNG**.

---

### **JUnit 4 Annotations**

* `@Test` тЖТ marks a test method.
* `@Before`, `@After` тЖТ run before/after each test.
* `@BeforeClass`, `@AfterClass` тЖТ run once before/after all tests.
* `@Ignore` тЖТ skip a test.

### **JUnit 5 Annotations (Jupiter)**

* `@Test` тЖТ marks test method.
* `@BeforeEach`, `@AfterEach` тЖТ run before/after each test.
* `@BeforeAll`, `@AfterAll` тЖТ run once before/after all tests.
* `@Disabled` тЖТ skip test.
* `@RepeatedTest` тЖТ repeat test multiple times.
* `@ParameterizedTest` тЖТ run test with different inputs.
* `@Tag` тЖТ group tests.
* `@DisplayName` тЖТ custom test name.

### **Mockito Annotations**

* `@Mock` тЖТ create mock object.
* `@Spy` тЖТ create spy (partial mock).
* `@InjectMocks` тЖТ inject mocks into tested object.
* `@Captor` тЖТ capture arguments passed to mocks.

### **TestNG Annotations**

* `@Test` тЖТ marks test method.
* `@BeforeMethod`, `@AfterMethod` тЖТ run before/after each test method.
* `@BeforeClass`, `@AfterClass` тЖТ run once before/after all test methods in class.
* `@BeforeSuite`, `@AfterSuite` тЖТ run once before/after entire suite.
* `@BeforeTest`, `@AfterTest` тЖТ run before/after `<test>` section in XML.

---

## ЁЯзй **Example (JUnit + Mockito)**

```java
import org.junit.jupiter.api.*;
import org.mockito.*;

import static org.mockito.Mockito.*;

class Calculator {
    int add(int a, int b) { return a + b; }
}

class CalculatorService {
    private Calculator calculator;

    CalculatorService(Calculator calculator) {
        this.calculator = calculator;
    }

    int addNumbers(int a, int b) {
        return calculator.add(a, b);
    }
}

public class CalculatorTest {

    @Mock
    private Calculator calculator;

    @InjectMocks
    private CalculatorService service;

    @BeforeEach
    void init() {
        MockitoAnnotations.openMocks(this);
    }

    @Test
    void testAdd() {
        when(calculator.add(2, 3)).thenReturn(5);
        Assertions.assertEquals(5, service.addNumbers(2, 3));
    }
}
```

### тЬЕ **Simple**

1. WhatтАЩs the difference between `@BeforeEach` and `@BeforeAll`?
   ЁЯСЙ `@BeforeEach` тЖТ runs before every test.
   ЁЯСЙ `@BeforeAll` тЖТ runs once before all tests (must be static).

2. What is `@Mock` in Mockito?
   ЁЯСЙ Creates a fake object to simulate dependencies.

---

### ЁЯдФ **Tricky**

3. Difference between `@Mock` and `@Spy`?
   ЁЯСЙ

* `@Mock` тЖТ fake object, all methods return default values.

4. Can you use `@Test(expected=Exception.class)` in JUnit 5?
   ЁЯСЙ No тЭМ. That was in JUnit 4. In JUnit 5, use:

### ЁЯФе **Tough**

5. How does Mockito `@InjectMocks` work internally?
   ЁЯСЙ It creates an instance of the class and tries to **inject mocks into constructor, setters, or fields** (in that order).

6. Difference between JUnit and TestNG?
   ЁЯСЙ

* JUnit тЖТ simple, lightweight, widely used.
* TestNG тЖТ more powerful (parallel execution, dependency testing, data providers).

   ЁЯСЙ

# ЁЯЯв **Cycle 8 тАУ Validation, Lombok & JSON Annotations**

---

## ЁЯУЦ **Concept**

### **A. Bean Validation (Hibernate Validator / JSR-380)**

Used for validating input data in Spring Boot REST APIs.

* `@NotNull` тЖТ field cannot be null.
* `@Size(min, max)` тЖТ string/collection size.
* `@Min`, `@Max` тЖТ numeric range.
* `@Email` тЖТ must be a valid email.
* `@Pattern` тЖТ regex validation.
* `@Past`, `@Future` тЖТ date validation.
* `@Valid` тЖТ trigger validation on nested objects.

---

### **B. Lombok (Boilerplate Reduction)**

Lombok auto-generates getters, setters, constructors, builders, logs, etc.

* `@Getter`, `@Setter` тЖТ generates getters and setters.
* `@ToString`, `@EqualsAndHashCode` тЖТ auto-generate methods.
* `@Data` тЖТ combo of `@Getter + @Setter + @ToString + @EqualsAndHashCode + @RequiredArgsConstructor`.
* `@Builder` тЖТ builder pattern.
* `@AllArgsConstructor`, `@NoArgsConstructor`, `@RequiredArgsConstructor` тЖТ constructors.
* `@Slf4j` тЖТ logger.

---

### **C. JSON (Jackson / Gson)**

For JSON serialization/deserialization.

* `@JsonProperty` тЖТ rename JSON field.
* `@JsonIgnore` тЖТ skip field.
* `@JsonInclude` тЖТ include non-null/non-empty fields.
* `@JsonFormat` тЖТ format date/time.
* `@SerializedName` (Gson) тЖТ rename field for Gson.

---

## ЁЯзй **Example**

```java
import jakarta.validation.constraints.*;
import com.fasterxml.jackson.annotation.*;
import lombok.*;

@Data
@AllArgsConstructor
@NoArgsConstructor
class User {

    @NotNull
    private Long id;

    @Size(min = 3, max = 20)
    private String name;

    @Email
    private String email;

    @JsonProperty("is_active")
    private boolean active;

    @JsonIgnore
    private String password;

    @JsonFormat(pattern = "yyyy-MM-dd")
    private java.util.Date dob;
}
```

### тЬЕ **Simple**

1. What does `@NotNull` do?
   ЁЯСЙ Ensures a field is not null (validation fails otherwise).

2. Difference between `@JsonIgnore` and `@JsonProperty`?
   ЁЯСЙ

* `@JsonIgnore` тЖТ exclude field from JSON.
* `@JsonProperty` тЖТ include field (possibly with a different name).

---

### ЁЯдФ **Tricky**

3. Difference between `@Valid` and `@Validated`?
   ЁЯСЙ

* `@Valid` тЖТ from JSR-303, validates nested objects.
* `@Validated` тЖТ from Spring, supports validation **groups**.

4. If a field is `@NotNull` but you send empty string `""`, will it pass?
   ЁЯСЙ Yes тЬЕ. `@NotNull` only checks null, not empty. For empty check, use `@NotEmpty` or `@NotBlank`.

---

### ЁЯФе **Tough**

5. How does Lombok work internally?
   ЁЯСЙ Uses **annotation processing** at compile time (`javac` plugin) to generate boilerplate code into `.class` files.

6. How to format a date field in JSON response?
   ЁЯСЙ Use `@JsonFormat(pattern="yyyy-MM-dd")`.

7. How do you validate nested objects?
   ЁЯСЙ Use `@Valid` on the parent field:

```java
class Order {
```

   ЁЯСЙ Configure `ObjectMapper`:

```java
objectMapper.setSerializationInclusion(JsonInclude.Include.NON_NULL);
```

Or use `@JsonInclude(Include.NON_NULL)` on class level.

# ЁЯФ╡ **Cycle 11 тАУ Spring MVC, Spring Boot, Cache & Session**

## ЁЯУЦ **Concept**

### **1. Spring MVC**

* Built on **DispatcherServlet** front controller.
* Key flow: **DispatcherServlet тЖТ HandlerMapping тЖТ Controller тЖТ ViewResolver тЖТ Response**.
* Uses annotations: `@Controller`, `@RestController`, `@RequestMapping`, etc.

### **2. Spring Boot**

* Simplifies configuration with `@SpringBootApplication`.
* Provides **auto-configuration**, **starter dependencies**, and **embedded server** (Tomcat/Jetty/Undertow).

### **3. Caching in Spring**

Spring supports **3 levels of caching (commonly discussed in interviews):**

* **Level 1: In-Memory (per request/session context)**

  * `@SessionAttributes`, `HttpSession`.
  * Stored in JVM memory during request/session lifecycle.

* **Level 2: Application-wide cache (Spring Cache Abstraction)**

  * Enable with `@EnableCaching`.
  * Use `@Cacheable`, `@CachePut`, `@CacheEvict`.
  * Backed by providers like ConcurrentMap, Ehcache, Redis.

* **Level 3: Distributed Cache (External Systems)**

  * Redis, Hazelcast, Memcached.
  * Shared across multiple app instances.
  * Prevents cache duplication in microservices.

### **4. Session Handling in Spring MVC/Boot**

* **HttpSession**: Default, managed by servlet container.
* **Spring Session**: External session management (e.g., Redis-backed).
* Scoped beans: `@SessionScope`, `@RequestScope`.

ЁЯУЦ Concept
1. JVM Memory Model (Runtime Data Areas)

Method Area (Metaspace in Java 8+) тЖТ stores class metadata, static variables, constant pool.

Automatic memory management.

3. Classloaders

Responsible for loading .class files into JVM.

Bootstrap ClassLoader тЖТ loads core Java classes (rt.jar).

Extension (Platform) ClassLoader тЖТ loads JDK extensions (lib/ext).

Application (System) ClassLoader тЖТ loads classes from classpath.

Custom ClassLoader тЖТ user-defined, can load classes dynamically.

ЁЯСЙ Works on Parent Delegation Model: request goes to parent first before child.

4. JIT (Just-In-Time Compiler)

Converts bytecode тЖТ native machine code at runtime for performance.

Types:

C1 (Client JIT) тЖТ optimized for startup time.

C2 (Server JIT) тЖТ optimized for long-running performance.

Works with HotSpot JVM (detects тАЬhot methodsтАЭ тЖТ compiles).

ЁЯЯв GC (Garbage Collection) Life Cycle & Methods
ЁЯУЦ GC Life Cycle

The JVM Garbage Collector manages object memory automatically.

Object Creation тЖТ new object is allocated in Heap.

ЁЯУЦ Methods related to GC

Runtime.getRuntime().gc() тЖТ Another way to request GC.

finalize() тЖТ called before object is destroyed (deprecated in Java 9+, replaced with Cleaner).

## ЁЯФ╣ **Core Java & JVM**

1. **What happens if you mark a method with `@Override` but the parent doesnтАЩt have it?**
   ЁЯСЙ Compile-time error.

2. **How does JVM decide which objects to GC?**
   ЁЯСЙ Objects unreachable from **GC Roots** (active threads, static refs, JNI).

3. **If two classes with same name are loaded by different ClassLoaders, are they equal?**
   ЁЯСЙ No тЭМ JVM treats them as different types.

4. **Difference between Heap & Stack?**
   ЁЯСЙ Heap тЖТ shared objects, Stack тЖТ thread-local method frames.

## ЁЯФ╣ **Multithreading & Concurrency**

5. **Difference between `volatile` and `synchronized`?**
   ЁЯСЙ

* `volatile` тЖТ ensures visibility, no atomicity.
* `synchronized` тЖТ visibility + atomicity + lock.

6. **How does `ConcurrentHashMap` achieve thread-safety?**
   ЁЯСЙ Java 7 тЖТ segmented locks. Java 8+ тЖТ striped locks + CAS.

7. **Can `Thread.sleep()` release lock?**
   ЁЯСЙ No тЭМ. `wait()` releases lock, but `sleep()` does not.

## ЁЯФ╣ **Spring Core & Boot**

8. **What happens if two beans of same type exist with `@Autowired`?**
   ЁЯСЙ Throws `NoUniqueBeanDefinitionException`. Fix тЖТ `@Qualifier` or `@Primary`.

9. **Difference between `@RestController` and `@Controller`?**
   ЁЯСЙ `@RestController = @Controller + @ResponseBody` тЖТ JSON response directly.

10. **How does `@Transactional` work internally?**
    ЁЯСЙ Uses **AOP proxies** around methods. Rollbacks happen only for **unchecked exceptions** unless configured.

11. **Difference between `@Value` and `@ConfigurationProperties`?**
    ЁЯСЙ

* `@Value` тЖТ injects single property.
* `@ConfigurationProperties` тЖТ maps multiple grouped properties.

## ЁЯФ╣ **Spring Data JPA & Hibernate**

12. **Difference between `get()` and `load()`?**
    ЁЯСЙ

* `get()` тЖТ hits DB immediately, returns `null` if not found.
* `load()` тЖТ lazy proxy, throws exception if not found.

13. **If you forget `mappedBy` in `@OneToMany`, what happens?**
    ЁЯСЙ Hibernate creates a **join table**.

14. **Difference between First-Level and Second-Level Cache in Hibernate?**
    ЁЯСЙ

* First-level тЖТ Session scope, mandatory.
* Second-level тЖТ SessionFactory scope, optional.

15. **WhatтАЩs the Saga Pattern?**
    ЁЯСЙ Distributed transaction strategy using local transactions + compensating rollbacks.

## ЁЯФ╣ **Spring Security**

17. **What happens in `@PostAuthorize`?**
